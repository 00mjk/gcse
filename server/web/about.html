Motivation

When I began to code in Go, I loved it. Directly importing from other open source projects is very useful. Projects are connected in a very elegant way.

So many people are contributing to Go society, and wheels are inevitably invented for several times. Choosing the one to import becames a problem. This is what a search engine can help.

A specific search engine designed for Go can benefit in the following ways:

  • By analyzing the importing relations between projects, popular projects can be found. This gives a better ranking than general search engine.
  • Structured data for each package can be extracted: package path, name, synopsis, documentation. Match rank can be calculated in a better way.
  • Stars (or watchers) of some sites are crawled to further help ranking.

Project

This is an open source project hosted on github.

If you find a bug, have a new idea or any other suggestion, please fire an issue at here.

Constructing GCSE, several sub-projects are published as well. Unlike GCSE itself, they are designed to be used by other projects.

  • go-index: a text indexing package, serving the index/search function in GCSE.
  • go-rpc: an RPC service through HTTP protocol.
